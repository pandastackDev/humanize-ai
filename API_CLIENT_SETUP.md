## 🔧 Configuration

### Environment Variables

**Local Development:**

```env
NEXT_PUBLIC_PYTHON_API_URL=http://localhost:8000
```

**Staging:**

```env
NEXT_PUBLIC_PYTHON_API_URL=https://humanize-python.vercel.app
```

### Scripts

**Generate API Client:**

```bash
# From packages/api-client/
pnpm generate         # Uses current OPENAPI_URL env or defaults to localhost
pnpm generate:local   # Generate from local backend
pnpm generate:staging # Generate from staging backend
```

## 📝 Usage Examples

### Server Component (Static Data)

```typescript
import { Sdk } from '@humanize/api-client';

const sdk = new Sdk();

export default async function Page() {
  const { data } = await sdk.getSampleDataApiV1ItemsDataGet({
    cache: 'force-cache',
    next: { revalidate: 60 }
  });

  return <div>{/* render data */}</div>;
}
```

### Client Component (Interactive)

```typescript
'use client';

import { useQuery } from '@tanstack/react-query';
import { getSampleDataApiV1ItemsDataGetOptions, Sdk } from '@humanize/api-client';

// Initialize SDK instance
new Sdk();

export function MyComponent() {
  const { data } = useQuery(getSampleDataApiV1ItemsDataGetOptions());

  return <div>{/* render data */}</div>;
}
```

## 🔄 Regenerating the Client

Whenever the backend API changes:

1. Ensure backend is running
2. Run `pnpm generate:local` (or `generate:staging`)
3. New endpoints and types are immediately available
4. TypeScript will catch any breaking changes

## 🐛 Troubleshooting

### Environment Variable Not Found

- Ensure `.env.local` exists in `apps/next/`
- Restart Next.js dev server after adding env vars
- Use `NEXT_PUBLIC_` prefix for client-side access

### Client Generation Fails

- Check backend is running and accessible
- Verify `/openapi.json` endpoint returns valid JSON
- Check network/CORS configuration

### Type Errors After Regeneration

- Run `pnpm install` from project root
- Restart TypeScript server in your IDE
- Clear Next.js cache: `rm -rf .next`
